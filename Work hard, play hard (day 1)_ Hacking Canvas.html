Work hard, play hard (day 1)

Inspired by cheesy corporate mottos, we'll use Laravel to build a small vacancies (=jobs) platform.

Requirements

  • Apache to run your local server
  • PHP to process your files
  • Terminal to run commands
  • Node and npm (for JS packages)
  • Composer to manage your dependencies (think of it like npm for PHP)

⚠️ We are using the word vacancy to avoid using job, because it's being used by some of Laravel's systems.
There won't be a direct conflict for basic projects, but it would cause a potential conflict should you decide to use Laravel's queue system later.
The queue system allows you to schedule tasks that take a bit longer to process.

Preparation

  • In your terminal, navigate to the folder where you want to start your new project
  • Follow the steps Koppelingen naar een externe site. here. Mind the following:
    • Change example-project to work-hard-play-hard (the name of our project)
    • Most default settings are fine: not starter kit, no git repo (your exercises folder already is one)
  • Use XAMP or php artisan serve to run the project
  • Check if you can see the welcome page (on XAMP, go to the /public folder of your project)

The project

Home

Find the home view in resources > views > welcome.
Tweak its contents t o only contain the following:

  • An h1 with the name of the recruitment agency: work hard, play hard (who doesn't like a hollow slogan?) - or something else if you're feeling creative (vacancies in the wizarding world? you name it!)
  • Update the <title> tag accordingly
  • Write a small <p> introducing the somewhat quirky agency

The vacancy overview

First, create a new view in resources > views > vacancy > index.blade.php.
Give it some contents: a title, and a couple of vacancy listings. The html is up to you.

Next, use the artisan command to create a new controller: php artisan make:controller VacancyController.
Inside, create a function index to load that view using return view('vacancy.index').

Finally, add a new route for /vacancies in routes > web.php that will run the VacancyController index function:
Route::get('vacancies', [\App\Http\Controllers\VacancyController::class, 'index']);

Give the route a name using ->name('vacancies.index').
This is a unique identifier to generate links in an easy way.
Time to give it a try: on the homepage, add a link to the vacancy overview.
Use {{ route('vacancies.index') }} to generate the correct url.

Shared components

Typically, an applications has a layout view containing generic code (head section, header and/or footer, ...).
Laravel has two main ways to do this: template inheritance (recommended) or components.
Read the Laravel docs on "building a layout". For our general structure, we'll go with template inheritance.
Provide a layout file containing the base html: layouts > app.blade.php.

Inside of layouts, create an extra folder _partials. Inside, add two views: one for your header and one for the footer.
Use @include to include them in the layout, so they are shown on every page.

A vacancy needs a detail page

Provide a page (vacancies/show.blade.php) for vacancy details following the same steps as the vacancy overview.
The content does not need to be dynamic, so one static detail page is sufficient.
The uri is /vacancies/show. Use show as name for the controller function and the view.
Make the vacancy titles clickable on the vacancy overview page.

Styling using Vite

CSS and JS often need some compiling (to minimise for production, or to go from TypeScript to JS).
Previously, this was done by tools based on Webpack.
Currently, Laravel Vite is popular for it's high speed.

Locate the vite.config.js file in the root of your project.
The default config will do for now: it will include one CSS and one JS file.
Search the Laravel docs to find out how you can load these files in your layout Blade file.

Change the CSS file to change the body background color.
First, install dependencies using npm install.
For development, we can use the command npm run dev to compile.
Run this now and make sure you can see the changes.
Finish this step by providing some basic styling for your project.
